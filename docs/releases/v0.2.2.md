# v0.2.2

Release date: 2026-07-24

## Summary

v0.2.2 is a compatibility maintenance release for the Codex plugin package and
repository marketplace. It adds approval-gated sandbox governance, optional
Decision Map planning, and a safer updater path contract for direct Codex
user-level skill installations.

## Included Skills

- `long-horizon-engineering`
- `ai-video-production`

## What Changed

- Added proposal-only local voice-tool and external 3D asset-provider sandbox
  governance.
- Added locked Skill Incubator capability research and routing safeguards.
- Added a locked public equity research sandbox that forbids customer-data
  upload, trading execution, and automated financial advice.
- Added optional Decision Map and deterministic Frontier planning support for
  long-running engineering work.
- Fixed `update_installed_skill.py` so project-scoped updates continue to use
  `--target-root <project>` while direct user-level Codex installs use
  `--target-skill-dir ~/.codex/skills/<skill>`.
- Rejected ambiguous updater targets, duplicate `~/.codex/.agents/skills`
  layouts, multi-skill apply operations, and mismatched direct skill
  directories.
- Kept `ai-video-production` behavior stable while aligning package metadata
  with the v0.2.2 release.

## Safety Model

- Updates are never automatic.
- Check-only remains the default.
- Applying an update requires explicit `--apply`.
- Apply mode requires exactly one explicit `--skill`.
- `--target-root` remains project-scoped and resolves to
  `<target-root>/.agents/skills/<skill>`.
- `--target-skill-dir` must point directly at an existing `skills/<skill>`
  directory containing `SKILL.md`.
- The updater rejects `~/.codex` as `--target-root` to avoid creating the
  unsupported duplicate path `~/.codex/.agents/skills/<skill>`.
- External providers, MCP setup, OAuth, uploads, downloads, trading, and runtime
  execution remain outside this release.

## Commands

Check only:

```bash
python3 scripts/skill_update_selfcheck.py
```

Project-scoped dry-run:

```bash
python3 .agents/skills/long-horizon-engineering/scripts/update_installed_skill.py \
  --target-root /path/to/project \
  --skill long-horizon-engineering
```

Direct Codex user-level dry-run:

```bash
python3 .agents/skills/long-horizon-engineering/scripts/update_installed_skill.py \
  --target-skill-dir ~/.codex/skills/long-horizon-engineering \
  --skill long-horizon-engineering
```

Apply only after review and explicit approval:

```bash
python3 .agents/skills/long-horizon-engineering/scripts/update_installed_skill.py \
  --target-skill-dir ~/.codex/skills/long-horizon-engineering \
  --skill long-horizon-engineering \
  --apply
```

## Rollback Behavior

Apply mode creates a timestamped backup before replacement. If post-update
validation fails, restore from the printed backup path. Backups are not deleted
automatically.
