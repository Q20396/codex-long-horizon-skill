# v0.2.1

Release date: 2026-07-15

## Summary

v0.2.1 is a maintenance release that fixes validation for user-level Codex
skill installations.

## Included Skills

- `long-horizon-engineering`
- `ai-video-production`

## What Changed

- Fixed `doctor.py` for `~/.codex/skills/<skill>/` installations.
- Fixed `check_skill_package.py` to auto-detect source-package and user-level
  skill layouts.
- Added a regression test that copies both bundled skills into a temporary
  user-level skills root and runs both validators.
- Kept the plugin package and repository marketplace metadata aligned with
  v0.2.1.

## Safety Model

- Check-only remains the default.
- Updates are never automatic.
- Codex-mediated online checks require explicit user approval.
- Applying an update still requires `--apply` and typed confirmation.
- Updates remain limited to approved bundled skills, with path and symlink
  safety checks.
- A timestamped backup is created before replacement, with rollback guidance
  printed after the operation.

## Commands

Check only:

```bash
python3 scripts/skill_update_selfcheck.py
```

Apply after review and typed confirmation:

```bash
python3 scripts/skill_update_selfcheck.py --apply
```

## Rollback Behavior

Apply mode creates a timestamped backup before replacement. If post-update
validation fails, the updater restores from that backup where possible and
prints the rollback location and command.

- Skill package checks passed for `long-horizon-engineering` and
  `ai-video-production` in source-package and user-level layouts.
- Trigger fixtures, description audit, safety audit, unit tests, release
