# v0.2.0

Release date: 2026-06-18

## Summary

v0.2.0 adds a safe, check-only-by-default update self-check flow for installed
Codex skills.

## Included Skills

- `long-horizon-engineering`
- `ai-video-production`

## What Changed

- Added v0.2.0 metadata to bundled skills.
- Added Safe Update / Self-Check Protocol to both `SKILL.md` files.
- Added customer-facing update prompt templates.
- Added release manifests under `releases/`.
- Added a dependency-free repo-level update self-check script.
- Added tests for update comparison, apply safety, traversal rejection, unknown
  skill rejection, and symlink safety.
- Restricted update targets to approved bundled skills only.
- Kept the update flow manual, backup-first, and rollback-documented.

## Safety Model

- Check-only by default.
- No automatic install or replacement.
- Codex-mediated use asks before online checking.
- Applying updates requires `--apply` plus typed confirmation.
- Unsafe paths, path traversal, unknown skill IDs, and unsafe symlinks are
  rejected.
- A timestamped backup is created before replacement.
- Rollback instructions are printed after update.
- If validation fails, restore from backup where possible.
- Updates are never automatic.

## Commands

Check only:

```bash
python3 scripts/skill_update_selfcheck.py
```

Apply after review and typed confirmation:

```bash
python3 scripts/skill_update_selfcheck.py --apply
```

## Rollback Behavior

Apply mode creates a timestamped backup before replacement. The self-check
prints the backup location and rollback command after update. If validation
fails during replacement, the updater attempts to restore from the backup.
