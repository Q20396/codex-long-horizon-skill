# Community Skills

This registry is for community skill submissions that may be reviewed later
without mixing unreviewed content into the core package.

## Submission Requirements

Submitters should provide:

- Public repository link
- License
- Short description
- Target use case
- Validation method
- Safety notes
- Maintainer name or handle

## Quality Requirements

- Includes a required `SKILL.md`.
- Has trigger-focused metadata.
- Includes example prompts.
- Describes validation workflow.
- Explains failure recovery.
- Keeps long guidance in `references/` or `templates/` when useful.

## Safety Requirements

- No secrets, API keys, private client data, legal evidence, family information,
  financial account details, identity documents, or confidential source content.
- No automatic merge, deployment, publication, upload, or destructive behavior
  without explicit user approval.
- Clear licensing and attribution expectations.
- External tool use must be optional and approval-gated.

## Review Process

1. Maintainers review the repository, license, docs, and examples.
2. Maintainers run relevant validation or request validation evidence.
3. Security, privacy, and licensing concerns are documented.
4. Approved entries may be listed below.
5. Proposed entries remain clearly separate from approved entries.
