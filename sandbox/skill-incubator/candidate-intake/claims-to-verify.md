# Claims To Verify

## Trust Boundary

Customer screenshots, short videos, stars, downloads, rankings, marketing
copy, and project names are candidate leads only. None is local evidence.

## Unverified Claims

- Any star count, download count, token-saving percentage, speed increase,
  cost reduction, accuracy, privacy, production-readiness, official status,
  agent count, model count, style count, success rate, or Top 10 ranking.
- Any claim that a named project is the unique source for Ralph Loop, UI UX Pro
  Max, MCP Builder, a PPTX skill, or a code-review skill.
- All repository default branches, commit SHAs, licenses, maintenance state,
  installation behavior, and telemetry claims for the ten newly named GitHub
  candidates. Their only verification attempt received HTTP 403.

## Required Evidence Before Future Evaluation

1. Read-only public repository metadata with a 40-character immutable commit.
2. Repository, file, model, asset, and dataset license review.
3. Explicit data-flow, filesystem, network, credential, service, cost, and
   background-process review.
4. A clean-room experiment with no third-party code import.
5. Baseline and regression evidence evaluated under the existing hard gates.
