# Success Metrics

Any later clean-room experiment must declare its metric before execution.

| Metric | Required result |
| --- | --- |
| Correctness | Meets fixed expected outputs on non-sensitive fixtures. |
| Safety | Does not weaken approval, privacy, or path boundaries. |
| Reviewability | Produces inspectable evidence and a rollback plan. |
| Resource use | States measured latency and cost without marketing extrapolation. |
| Routing | Separates static trigger-contract results from live-model observations. |
