# Experiment Proposal

## Experiment ID
`MAD-SKILL-011`
## Name
Industrial Skill Compiler and Evaluator
## Status
`locked`
## Problem Statement
Candidate Skills need consistent trigger, permission, data-flow, threat, evaluation, fixture, rollback, and promotion-readiness evidence before any change is considered.
## Why This Is Relevant
A compiler-like review contract can turn an idea into auditable artifacts without modifying the original Skill.
## Source Patterns
Customer-provided industrial governance pattern; no third-party source commit is claimed.
## Clean-room Boundary
Use the stated abstract requirements only; copy no external code or proprietary procedure.
## Explicit Non-goals
No automatic Skill modification, trigger editing, installation, update, execution, or promotion.
## Inputs
Future customer-approved candidate description and exact approved target paths.
## Outputs
Future trigger-contract, permission-contract, data-flow, threat-model, eval-plan, positive/negative/ambiguous fixtures, baseline plan, rollback plan, and promotion readiness review.
## Permissions Required
Review materials are not permission to read or write target Skills.
## Network Requirements
None for design.
## Dependency Requirements
None for design.
## Cost Possibility
Future evaluation may consume compute; no cost occurs here.
## Customer Approval Gates
Approve target paths, fixtures, evaluation, implementation, and promotion separately.
## Expected Benefits
More consistent review before mutation or release.
## Failure Modes
False assurance, fixture overfitting, approval confusion, incomplete threat model, or unsupported claim.
## Baseline Comparison
Ad hoc Skill editing with no unified evidence package.
## Required Hard Gates
All gates; human control, source/license integrity, privacy, and rollback are critical.
## Evaluation Categories
Contract completeness, fixture coverage, safety, permission clarity, and baseline quality.
## Promotion Constraints
`core-policy-candidate` is only an initial layer suggestion; remain locked without separate evidence and customer approval.
## Rejection Conditions
Reject automatic target modification, hidden scope expansion, or promotion without explicit evidence.
## Open Questions
What minimum evidence package remains useful without making simple Skills too heavy?
