# Licensing Notes

This is a lightweight engineering guide, not legal advice.

## Learn Patterns, Do Not Copy

Public GitHub projects are useful for studying patterns, architecture, workflow design, and tradeoffs. Do not assume their code, docs, examples, templates, assets, or branding can be copied.

Allowed for this skill:

- Original summaries
- Generalized workflow ideas
- Architecture concepts
- High-level comparisons
- Fresh templates written for this repository

Not allowed:

- Copying repository source code
- Copying large README sections
- Reusing project-specific templates
- Reusing assets, logos, fonts, music, or voices without permission
- Removing attribution from third-party work

## Check Licenses Before Reuse

Before using external code or assets, check:

- Repository license
- Package license
- Asset license
- Model license
- Font license
- Music and sound license
- Voice or likeness permission
- Commercial-use restrictions
- Attribution requirements
- Share-alike or copyleft obligations

## Remotion Caution

Remotion has special licensing considerations, including company-license requirements for some organizations. Check the current Remotion license and commercial terms before adopting Remotion itself.

This repository's `ai-video-production` skill is original documentation inspired by general patterns and does not include Remotion code.

## HyperFrames Caution

HyperFrames is public and open source, but that does not mean its templates, examples, or project prose should be copied into this repository. Study the HTML-native, preview-first, lint-before-render, manual-render-gate pattern and write original artifacts.

## Image And Video Generation Caution

Generated media can still have usage constraints. Check:

- Provider terms
- Model license
- Input asset rights
- Publicity and likeness rights
- Brand and trademark concerns
- Platform policies
- Whether generated outputs can be used commercially

## Original Minimal Examples

If code is needed in a future project, write original minimal examples that implement the required idea without copying external code. Keep examples scoped, attributed where appropriate, and separated from third-party project branding.
