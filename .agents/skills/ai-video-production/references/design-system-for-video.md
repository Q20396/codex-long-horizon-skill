# Design System for Video

Use this protocol when repeatable or branded video work needs visual consistency
before generating prompts, storyboards, or shot lists.

The design system should act as a source of truth for style choices, not as
permission to clone protected brand systems.

## Fields

- brand context
- audience
- visual mood
- color palette
- typography
- composition
- lighting
- camera language
- motion language
- transition rules
- lower thirds / captions
- logo handling
- forbidden patterns

## Workflow

1. Create or read `DESIGN.md`.
2. Extract visual tokens.
3. Generate style-consistent shot prompts.
4. Run visual consistency QA.
5. Prepare client handoff.

## Rights and Brand Safety

Use brand/design inspiration as reference, not a direct clone, unless the user
owns the rights or is doing internal analysis.
