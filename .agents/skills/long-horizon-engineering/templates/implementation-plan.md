## Task Granularity Rules

Each task should be small enough to review independently.

A good task includes:
- exact files to create or modify
- exact behavior or documentation change
- validation command
- expected result
- rollback note

Avoid vague tasks:
- "improve docs"
- "clean up scripts"
- "make tests better"
- "refactor everything"
