## Keyboard And Focus

- [ ] Interactive elements are reachable by keyboard.
- [ ] Focus order follows the visual and task order.
- [ ] Focus states are visible.
- [ ] Keyboard shortcuts do not block browser or assistive technology defaults.

## Names And Semantics

- [ ] Buttons and controls have accessible names.
- [ ] Form fields have labels or equivalent accessible descriptions.
- [ ] Headings follow a meaningful hierarchy.
- [ ] Icons are labelled when meaningful and hidden when decorative.
- [ ] Tables, lists, landmarks, and dialogs use appropriate semantics.

## Visual Access

- [ ] Text contrast is sufficient for its size and background.
- [ ] Important information is not conveyed by color alone.
- [ ] Text can wrap without overlap or clipping.
- [ ] Zoom and responsive layouts preserve core functionality.

## Motion And Interaction

- [ ] Motion is purposeful and not excessive.
- [ ] Reduced-motion preferences are respected when applicable.
- [ ] Touch targets are large enough for mobile use.
- [ ] Hover-only interactions have keyboard or touch alternatives.

## Forms And Errors

- [ ] Errors identify the affected field.
- [ ] Error messages explain the fix.
- [ ] Required fields are clear.
- [ ] User input is preserved after validation failures where reasonable.
