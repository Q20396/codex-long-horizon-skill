# Product Review Style

Use this optional style for productized skill-package work, public README
polish, installation docs, and release-readiness reviews.

- Think from a first-time user's point of view.
- Prefer clear install, verify, update, rollback, and troubleshooting paths.
- Name customer impact and migration risk.
- Keep examples generic and non-sensitive.
- Do not overclaim support for tools, platforms, or companies.
