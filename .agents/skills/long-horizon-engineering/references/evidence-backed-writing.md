# Evidence-Backed Writing Protocol

Use this protocol for technical reports, research summaries, PR descriptions,
legal/evidence summaries, academic-style writing, investor or market analysis,
and customer-facing claims.

## Principles

- Every material claim needs evidence or a clear assumption label.
- Do not overstate conclusions beyond the evidence.
- Separate facts, inference, assumption, and recommendation.
- Preserve uncertainty.
- Prefer primary sources.
- For papers, track title, authors, date, venue, and source.
- For code, cite file paths, tests, command outputs, and validation logs.

## Claim-Evidence Workflow

1. Extract material claims.
2. Classify each claim type.
3. Attach evidence.
4. Mark confidence.
5. Flag gaps.
6. Rewrite unsupported claims.

## Reviewer Mindset

Ask:

- What would a skeptical reviewer challenge?
- What evidence is missing?
- Which claim should be weakened?
- Which citation or file reference does not actually support the sentence?
